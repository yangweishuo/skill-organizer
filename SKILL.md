# skill-organizer：智能 SKILL.md 整理优化器

## 一、概述

`skill-organizer` 是一个**元技能**——操作对象是其他技能的 SKILL.md 文件。它将过往多次 SKILL.md 优化实践中验证的技术系统化为可复用的检测-修复管道，并具备自进化能力：每次运行后根据执行结果调整检测策略，使得后续运行越来越精准。

### 适用场景

| 场景 | 说明 |
|------|------|
| 初次整理 | 对新建或未整理过的 SKILL.md 做全量检测修复 |
| 增量维护 | 插入新章节后运行，检查是否引入编号错误或层级冲突 |
| 定期巡检 | 作为定时任务，每周自动扫描所有技能的 SKILL.md |
| 版本升级 | 升级 version 字段后运行 D7 检测器核验一致性 |

## 二、管道架构

```
SKILL.md → Parser → 结构树 JSON
                       ↓
            10 个 Detector 并行扫描 → 问题清单（按严重度分级）
                                          ↓
                               Planner → 修复计划（auto / manual 分流）
                                            ↓
                                  Executor → 修复后文件 + 执行日志
                                                ↓
                                     Learner → 更新规则权重 + 追加新模式
```

### 模块职责

| 模块 | 输入 | 输出 | 说明 |
|------|------|------|------|
| Parser | SKILL.md 原始文本 | 结构树 JSON | 提取标题层级、YAML 元信息、代码块、内部引用 |
| Detector | 结构树 JSON | 问题清单 | 10 个检测器并行扫描，每个问题带严重度和修复建议 |
| Planner | 问题清单 | 修复计划 | 按严重度分流：Bug 类自动执行，结构类排队确认 |
| Executor | 修复计划 | 修复后文件 + 日志 | 使用 `edit_file` 精确替换，每次操作前自动备份 |
| Learner | 执行日志 | 更新后的规则/权重 | 分析命中率，调整检测策略，追加新规则 |

## 三、10 类检测器

### Bug 类（自动修复）

| 编号 | 检测器 | 检测逻辑 | 修复策略 |
|------|--------|----------|----------|
| D1 | 版本编号连续性 | 正则提取所有 `v\d+\.\d+(\.\d+)?` 编号，按章节上下文验证归属 | 修正错位编号，如 `v6.52.2` → `v6.53.2` |
| D3 | 标题层级冲突 | 提取 `## 一、` `## 二、` 等中文编号，与顶级章节编号做交叉比对 | 将冲突的 `##` 降级为 `###` |
| D7 | Version 一致性 | 比对 YAML `version` 值 vs 内容中出现的最新 `v\d+\.\d+` | 以内容中最新的为准修正 YAML |
| D10 | YAML 完整性 | 检查 `name` / `description` / `version` 三字段是否存在 | 补全缺失字段 |

### 结构类（需确认）

| 编号 | 检测器 | 检测逻辑 | 修复策略 |
|------|--------|----------|----------|
| D2 | 模块聚合度 | 按主题关键词分组章节（如"抖音"规则应聚合），检测被 >500 行无关内容隔开 | 移动章节块到同主题区域 |
| D5 | 重复内容 | 相邻章节间做 simhash 比对，相似度 >0.85 标记 | 保留一份，删除重复 |
| D6 | 章节编号一致性 | 检测中文编号（一~九）是否连续无跳跃 | 提示缺失或跳跃的编号 |

### 优化类（仅报告）

| 编号 | 检测器 | 检测逻辑 | 建议 |
|------|--------|----------|------|
| D4 | Description 精简度 | 检查是否线性堆砌版本号，字符数 >300 告警 | 精简为一句核心能力描述 |
| D8 | 代码块语言标记 | 检测 ` ``` ` 后是否缺少语言标记 | 补充语言标记 |
| D9 | 引用有效性 | 提取「见 v6.x 节」等内部引用，检查目标锚点 | 修正失效引用或删除 |

## 四、混合执行模式

```
┌──────────────────────────────────────────────┐
│              混合执行模式决策树               │
├──────────────────────────────────────────────┤
│  问题严重度 = Bug（D1/D3/D7/D10）？          │
│    ├─ 是 → 自动修复，输出修复摘要            │
│    └─ 否 → 严重度 = 结构（D2/D5/D6）？       │
│             ├─ 是 → 展示问题+建议，确认后执行 │
│             └─ 否 → 仅报告末尾列出建议       │
└──────────────────────────────────────────────┘
```

## 五、自进化机制

### 学习数据存储

学习数据写入 `rules/memory.json`，跨运行持久化：

```json
{
  "version": "1.0",
  "run_count": 0,
  "detectors": {
    "D1": {"weight": 0.90, "hits": 0, "false_positives": 0, "dormant": false},
    "D2": {"weight": 0.70, "hits": 0, "false_positives": 0, "dormant": false},
    "D3": {"weight": 0.85, "hits": 0, "false_positives": 0, "dormant": false},
    "D4": {"weight": 0.50, "hits": 0, "false_positives": 0, "dormant": false}
  },
  "custom_rules": [],
  "file_history": []
}
```

### 三种进化行为

| 行为 | 触发条件 | 效果 |
|------|----------|------|
| **权重调整** | 每次运行后自动执行 | 命中率高的检测器提升优先级；命中率 <30% 的自动降噪 |
| **规则追加** | 用户手动修正了未自动检测到的问题 | 将新模式追加为对应检测器的子规则，写入 `custom_rules` |
| **策略自适应** | 累计数据触发阈值 | 文件 >300KB 自动分段分析（5000行切片）；连续 3 次假阳性进入休眠 |

## 六、使用方式

### 命令行

```bash
# 分析模式（仅输出报告）
python scripts/main.py <SKILL.md路径> --dry-run

# 混合模式（默认）
python scripts/main.py <SKILL.md路径>

# 全自动模式
python scripts/main.py <SKILL.md路径> --auto

# 查看学习数据
python scripts/main.py --stats
```

### Agent 调度

当用户说「整理 XX 技能的 SKILL.md」时，Agent 应：

1. 确认目标文件路径
2. 运行 `python scripts/main.py <路径>`
3. 将 Planner 输出的结构类问题通过 `ask_user` 展示给用户
4. 用户确认后，Executor 执行修复
5. Learner 自动更新 `rules/memory.json`

## 七、文件结构

```
skills/custom/skill-organizer/
├── SKILL.md                        # 本文件
├── scripts/
│   ├── main.py                     # 主入口，管道调度
│   ├── parser.py                   # 结构树解析器
│   ├── detector.py                 # 10个检测器引擎
│   ├── planner.py                  # 修复计划生成器（auto/manual分流）
│   ├── executor.py                 # 修复执行器（edit_file操作）
│   └── learner.py                  # 自进化引擎
├── rules/
│   ├── memory.json                 # 学习数据持久化
│   └── patterns.json               # 检测模式定义
└── references/
    └── detection-catalog.md        # 检测规则详细说明
```

## 八、约束

- 只编辑目标 SKILL.md，不动其他文件
- 每次编辑前自动备份（追加 `.bak` 时间戳后缀）
- 使用 `old_str` / `new_str` 精确匹配替换，不依赖行号
- 单次运行最多执行 20 次编辑操作
- 禁止修改系统级核心路径下的文件
