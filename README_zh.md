# skill-organizer：智能 SKILL.md 整理优化器

## 项目概述

`skill-organizer` 是一个**元技能**——它以其他技能的 SKILL.md 文件为操作对象，将过往多次 SKILL.md 优化实践中验证的技术系统化为可复用的检测-修复管道。每次运行后，它还会根据执行结果调整检测策略，实现持续自进化。

### 核心理念

SKILL.md 文件随功能迭代不断膨胀（如 web-presentation-video 已达 381 KB / 14675 行），手工维护章节结构、版本编号和标题层级极易出错。skill-organizer 将这种维护工作自动化，让人类从机械的结构检查中解放出来。

## 项目背景

### 问题起源

在对 web-presentation-video 的 SKILL.md 进行迭代优化时，我们诊断出四类典型问题：

| # | 问题 | 严重度 | 案例 |
|---|------|--------|------|
| 1 | 版本编号错位 | 致命 | `v6.52.2` 编号在编辑 v6.53 节时未同步更新，残留旧编号 |
| 2 | 模块碎片化 | 严重 | v6.53（抖音规则）与 v6.54（抖音中长视频）被 ~7700 行无关内容隔开 |
| 3 | 标题层级冲突 | 严重 | 子模块内部 `## 一、` 与顶级九大章编号重合 |
| 4 | Description 冗余 | 一般 | YAML description 线性堆砌版本号，长达 700 字符 |

手动修复了这些问题的同时，思考到这类维护场景会随着技能数量增加而频繁出现，于是决定将修复流程沉淀为可自进化的元技能。

## 架构设计

### 管道架构

```
SKILL.md → Parser → 结构树 JSON
                       ↓
            10 个 Detector 并行扫描 → 问题清单（三级严重度）
                                          ↓
                               Planner → 修复计划（auto / manual 分流）
                                            ↓
                                  Executor → 修复后文件 + 执行日志
                                                ↓
                                     Learner → 更新规则权重 + 追加新模式
```

### 模块职责

| 模块 | 输入 | 输出 | 核心逻辑 |
|------|------|------|----------|
| **Parser** | SKILL.md 原始文本 | 结构树 JSON | 正则提取标题层级、YAML 元信息、代码块边界、版本引用、内部引用 |
| **Detector** | 结构树 JSON | 问题清单 | 10 个独立检测器并行扫描，每个问题附带严重度、修复类型和建议 |
| **Planner** | 问题清单 | 修复计划 | 按严重度三级分流：Bug 自动 / 结构确认 / 优化报告 |
| **Executor** | 修复计划 + 原文件 | 修复后文件 + 日志 | `old_str`/`new_str` 精确匹配替换，编辑前自动备份 |
| **Learner** | 执行日志 + 文件信息 | 更新后的规则/权重 | 权重命中率驱动调整、多模式追加、自适应策略判断 |

## 10 类检测器

### Bug 级（自动修复）

| 编号 | 检测器 | 检测逻辑 | 修复策略 |
|------|--------|----------|----------|
| **D1** | 版本编号连续性 | 正则提取所有版本号，按章节上下文验证归属 | 修正错位编号 |
| **D3** | 标题层级冲突 | 提取中文编号标题，与顶级章节编号交叉比对 | 冲突的 `##` 降级为 `###` |
| **D7** | Version 一致性 | 比对 YAML `version` 值 vs 内容最大版本号 | 以内容为准修正 YAML |
| **D10** | YAML 完整性 | 检查 `name` / `description` / `version` 三字段存在性 | 补全缺失字段 |

### 结构级（需确认）

| 编号 | 检测器 | 检测逻辑 | 修复策略 |
|------|--------|----------|----------|
| **D2** | 模块聚合度 | 主题关键词分组后检测同主题章节间距 >500 行 | 移动章节块到同主题区域 |
| **D5** | 重复内容 | 相邻章节长度 >50 的行做集合交集，>3 行重复告警 | 保留一份删除重复 |
| **D6** | 章节编号一致性 | 中文编号序列连续性检查 | 提示跳跃或缺失 |

### 优化级（仅报告）

| 编号 | 检测器 | 检测逻辑 | 建议 |
|------|--------|----------|------|
| **D4** | Description 精简度 | 字符数 >300 或版本号 >5 个告警 | 精简为核心一句 |
| **D8** | 代码块语言标记 | 检测代码块是否缺少语言标记 | 补充语言标记 |
| **D9** | 引用有效性 | 提取内链，检查目标锚点是否存在 | 修正或删除失效引用 |

## 混合执行模式

```
问题严重度
├─ Bug（D1/D3/D7/D10）→ 自动修复，无需询问
├─ 结构（D2/D5/D6）   → 展示问题+建议，确认后执行
└─ 优化（D4/D8/D9）   → 仅列在报告末尾，不自动处理
```

## 自进化机制

### 三种进化行为

| 行为 | 触发条件 | 效果 |
|------|----------|------|
| **权重调整** | 每次运行后自动执行 | 命中率高的提升优先级；<30% 自动降噪 |
| **规则追加** | 手动修正了自动未检测的问题 | 新模式追加到对应检测器，写入 `custom_rules` |
| **策略自适应** | 累计数据触发阈值 | >300KB 自动分段（5000 行切片）；连续 3 次假阳性休眠 |

## 使用方式

### 命令行

```bash
# 分析模式（仅输出报告，不改文件）
python scripts/main.py <SKILL.md路径> --dry-run

# 混合模式（默认：Bug 自动修，结构确认后修）
python scripts/main.py <SKILL.md路径>

# 全自动模式（跳过确认）
python scripts/main.py <SKILL.md路径> --auto

# 查看学习数据
python scripts/main.py --stats
```

## 文件结构

```
skill-organizer/
├── SKILL.md                        # 技能主文件
├── scripts/
│   ├── main.py                     # 主入口（管道调度器）
│   ├── parser.py                   # 结构树解析器
│   ├── detector.py                 # 10 个检测器引擎
│   ├── planner.py                  # 修复计划生成器
│   ├── executor.py                 # 修复执行器
│   └── learner.py                  # 自进化引擎
├── rules/
│   ├── memory.json                 # 学习数据（权重/命中/休眠状态）
│   └── patterns.json               # 检测模式定义（正则/阈值/策略）
└── references/
    └── detection-catalog.md        # 检测规则详细说明文档
```
